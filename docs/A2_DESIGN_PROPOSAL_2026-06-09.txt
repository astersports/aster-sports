==============================================================================
PURPOSE
==============================================================================
The operator control surface that writes organizations.auto_notifications JSONB
— the only comms config that currently has NO UI (set via raw SQL today). Makes
operator-settable:
  - the A1 Stream-B enable flag (rsvp_nudges_enabled),
  - the Stream-A reminders toggle (reminders_enabled),
  - the going-floor (rsvp_min_going).
CLAUDE.md §16.5 names it "AutoNotificationSettingsSheet (Stream B -> 'Minimum
confirmed going')"; §8 says Stream A's reminders_enabled gates from it.

==============================================================================
PROPOSED CONTROLS  (all under auto_notifications, MERGED-write so sibling keys survive)
==============================================================================
1. Event reminders (Stream A) — toggle reminders_enabled, DEFAULT ON (!== false).
   Helper: "Families get a reminder 3 days, 1 day, and 4 hours before each game."
2. Short-roster RSVP nudges (Stream B) — toggle rsvp_nudges_enabled, DEFAULT OFF
   (=== true).
   Helper: "When a game is within 48 hours and fewer than [N] players have
   confirmed, we'll draft a nudge to your Radar to review and send. Nothing sends
   to families automatically."
3. Minimum confirmed going — stepper rsvp_min_going, default 5, range 1–15.
   DISABLED (greyed, not hidden) when #2 is off, so the operator sees the value
   they'll get when they enable it.
   Helper: "You need this many confirmed to field a game."

==============================================================================
CROSS-SURFACE INVARIANT  (AP#43 / AP#63 — the key design point)
==============================================================================
The form READS via the SAME helpers the cron handler uses — rsvpNudgesEnabled()
+ rsvpMinGoingThreshold() from the AP#30 threshold module
(src/lib/cron/rsvpNudgeThreshold.js) — so the UI's notion of on/off + floor can
never drift from what briefing-auto-draft-tick actually does. Reminders default
reuses the reminders_enabled !== false rule. New useOrgAutoNotifications() hook
wraps those for read + a merged save(patch) writer (read current JSONB, spread,
set the touched keys — never clobber other auto_notifications keys).
