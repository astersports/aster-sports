D. ARCHITECTURAL FORKS — NEED A RULING (labeled leans, AP#39)
================================================================================
  F1 — NON-SEASON (tryout/camp/clinic) REGISTRATION PATH.  [HEADLINE]
     REQUEST: how does a parent register for a program type that has no divisions?
     CONTEXT: live "Fall 2026 Tryouts" is a tryout (hasDivisions=false), funnel
       requires a division → parent cannot register today. b3-funnel render shows
       a tryout with a flat per-child fee ("Charlie · Fall 2026 Tryouts $45") and
       NO division picker — so the intended UX already implies a flat-fee path.
     OPTIONS:
       (A) Auto-create ONE implicit "default division" per non-season program at
           create-time (a flat-fee enrollment unit, gender=coed, no grade band).
           Funnel works UNCHANGED. Smallest code delta. Fees reuse division_fees.
       (B) Separate flat-fee registration path for non-season types (program-level
           fee, skip the division step). New flow + new fee source. More surface.
       (C) Tryouts use tryout_sessions/attendees (pick a session) — different model;
           tryout_sessions exist but are empty; doesn't match the b3-funnel render.
     CC LEAN: (A). It makes the live program registrable with the least new
       surface, keeps ONE funnel + ONE fee source (division_fees), and matches the
       b3-funnel render (the "division" line is just the program's flat unit). Needs
       a tiny admin step (a fee field for non-season programs) + the funnel to skip
       the visible division picker when there's exactly one implicit unit.
     RENDER NEEDED: non-season register flow (flat fee, no visible division pick) +
       the admin "tryout/camp fee" capture.

  F2 — pr1b RLS NARROWING TIMING.  [now-urgent]
     REQUEST: narrow registration_fees + player_equipment SELECT now, or hold?
     CONTEXT: documented deviation's precondition ("no parent UI") expired when
       B1/B2 shipped. Revert path is written and ready.
     OPTIONS: (A) apply pr1b now (admin OR own-child) — CC can apply under the lane.
       (B) hold until funnel registration actually goes live for parents.
     CC LEAN: (A) apply now. The leak is live the moment a parent can authenticate,
