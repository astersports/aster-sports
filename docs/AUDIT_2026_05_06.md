## P1 — High bugs (visible UX degradation, intermittent breakage)

### Async / data layer

- `useRsvps.fetch` no race-cancellation (rapid RSVP taps stomp); same in `useDuties`, `useComments`, `useCheckIns`
- `useActivities.refetch` flips `loading=true` even when cache exists → full screen wipe + scroll loss
- Module-level cache cross-account leak: `ChildRsvp.responseCache`, `useEventRsvpNotes`, `useActivities`, `useLocations`, `useTournaments`. Sign-out doesn't clear them.
- `useCreateActivity` non-atomic series + duty creation — duty-insert failure leaves orphan events
- `parentContext.fetchParentContext` keeps only first team per child (`rosters[0]?.team_id`) — **partially fixed in Wave 4** (now reads `parent_context_v` which returns all active teams)

### React correctness

- `useScoreDraft.performSave` retry timer never cancels on unmount (only `debounceRef`/`flashRef` cleaned)
- `SchedulePage` lazy-init reads URL once — `selectedTeam` sticky if URL changes via tap
- `RsvpPlayerRow.noteText` desyncs from server after save+refetch (local state stale)
- `NextUpCardMed` + `NextUpCardMax` run a second `setInterval` for countdown alongside `useNow()` — two clocks tick out of phase
- `useOrgBranding` cleanup-and-reapply storm — `useEffect([org])` keys on object identity; should depend on `org?.id`
- `ToastProvider` value re-creates each render; `PreferencesProvider.updatePreference` deps include `preferences` — cascading re-renders

### Schema / canonical-table drift

- `useEventRsvpCounts.js:28` reads `roster_members` for team size — §11.5 says `team_players` is canonical. No `left_at IS NULL` filter → inflates `noResponse` counts.
- `useAdminStats.js:66` — "Players" KPI reads `roster_members`, inflated by churned players
- `useRoster.js:14` — pulls every guardian's email + phone for every team load (PII overfetch)

### Form / UX-breaking

- `wizardForm.eventToForm` mixes UTC date-slice with local time-string — late-evening NY events shift 24h on re-save
- `PlayerRow.jsx:47` uses `player.attendance_pct || 85` — **fixed in this session** (now shows real data or nothing)
- `useFilteredRoster.js:27-29` numeric sort on text jersey numbers — "00","1","10","2" sorts unstably
- `Header.jsx:25` bell-tap "Notifications coming soon" — bait-and-switch
- `CoachHomePage.jsx:29-41` derives myTeams from events, not `team_staff` — coach with no upcoming events sees empty strip
- `MessageTeamFAB.jsx:13-14` — `onClick` is `navigator.vibrate?.(10); /* TODO */`. Permanently dead button.
- `ForgotPasswordPage.jsx:42-44` — "Password reset is coming soon." Dead-end for any parent.
- `MessagesPage.jsx` — 10-line stub; bottom-nav promotes nonexistent feature.
