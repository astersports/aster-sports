3. THE CORE ARCHITECTURAL PROBLEM — TWO DATA SOURCES, ONE CONCEPT (AP #63)
================================================================================
The .io/aau hub today runs on TWO data systems that don't reconcile:

  SOURCE A — the LEGACY tRPC/scraper (astersports-web's own server + scraper.ts).
    Scope: ONE team ("Aster AAU 11U Girls" = Charlie's Legacy 11U Girls team).
    Powers: Live Scores, History, Records, the hero stat bar, Film hero, Mission, Locations.
    Quality: RICH + REAL — full game log, PLACEMENTS (Champions/Finalists/Final Four),
    PPG/opponent-PPG, streaks. This is the better data for the team it covers.

  SOURCE B — the NEW cross-program BACKBONE (aster-sports Supabase public RPCs).
    Scope: ALL programs/tournaments ingested (incl. Rowan's Chris Ward; the 9 tournaments).
    Powers: the Standings tab + (held) the real-data My Kids view.
    Quality: REAL but THINNER — division_games is external-vs-external game results only;
    it has NO bracket/placement notion and NO box-score/PPG. It is the ONLY source that
    has Rowan (cross-program) and the full cross-program standings.

They disagree. Example [GROUNDED]:
    ZG Chase for the Chain NY, Charlie's Legacy Girls-5th:
      Source A (tRPC/History tab): 4-0, CHAMPIONS.
      Source B (get_public_team_season): 3-0.
    The gap is the bracket/championship game + the placement badge, which the legacy
    scraper captured (game_results) but the backbone's division_games does not yet model.

This is AP #63 (same concept, divergent source/scope) — the platform's dominant bug
pattern — now visible to the operator on screen. It is the central thing to resolve.

7. DECISIONS FOR THE ARCHITECT  (each: REQUEST · EVIDENCE · OPTIONS · CC LEAN)
================================================================================
D-HUB1 — SOURCE OF TRUTH per concept (the load-bearing fork).
  REQUEST: which source answers each tab, given Source A is richer for Charlie's team and
    Source B is the only cross-program source (Rowan)?
  EVIDENCE: §3 (4-0 vs 3-0), §4 (Rowan only in B).
  OPTIONS:
    (1) Per-kid routing NOW: Charlie's content tabs read the rich Source A; Rowan's read
        Source B; Standings auto-scopes either way. Ships the lens fast; two code paths.
    (2) Backbone-only: make Source B the single source for everyone; requires enriching B
        with bracket/placement + box-score (see D-HUB3) before it matches A. Cleaner end
        state; more build before Charlie's tabs look as good as today.
    (3) Status quo (team site + separate Standings): no kid lens. Rejected — contradicts
        the model.
  CC LEAN: (1) now → (2) as the convergence target. Route per kid this week so the lens
    ships on real data; in parallel enrich the backbone (D-HUB3) until it can be the one
    source and the legacy scraper retires. Avoids a long dark period and AP #63 churn.

D-HUB2 — FILM HOSTING + the fabrication fix.
  REQUEST: (a) remove the fabricated film NOW (no fork); (b) where does .io serve real
    per-kid clips?
  EVIDENCE: §5 Film. Clips currently live in the legacy-hoopers app's storage; the .io
    featured video is an unplayable Drive link.
  OPTIONS for (b): (i) Supabase Storage on the backbone (one home, public-read for public
    clips, gated for child-private per the invariant); (ii) keep clips in the per-program
    app + serve via signed URL; (iii) defer real video, ship AI-breakdown cards only.
  CC LEAN: do (a) immediately regardless. For (b): (i) Supabase Storage on the backbone —
    one source, fits the public/private gate; film is Track 2 (privacy posture first per
    the prior ruling), so (iii) AI-breakdown-only is the honest interim.

D-HUB3 — BACKBONE ENRICHMENT (the real fix for 4-0 vs 3-0).
  REQUEST: capture bracket results + placement (Champions/Finalist/…) and, eventually,
    box-score/PPG in the backbone so Source B can match Source A.
  EVIDENCE: §3. division_games skips bracket placeholders + has no placement/box-score.
  OPTIONS: (1) extend the ingest to record bracket_slots + a final placement per
    division-team; (2) also ingest box scores where TM exposes them (PPG).
  CC LEAN: (1) first (placement is what's visibly wrong); (2) later (PPG is nice-to-have).

D-HUB4 — NAMING / BRAND per program.
  REQUEST: the hub says "Aster AAU 11U Girls" everywhere; the lens should show the kid's
