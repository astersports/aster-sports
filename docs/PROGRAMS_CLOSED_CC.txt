3. SCOPE LINE — WHAT STAYS WHERE (records/standings exclusion)
------------------------------------------------------------------------------
Records / standings exclusion stays TEAM-LEVEL (isCompetitiveTeam, keyed off the
team_type), NOT folded into the program registry. This is deliberate and was held
through the build: a single program can carry both competitive and
non-competitive teams, so the exclusion decision belongs at the team, not the
program. The registry's per-type `competitive` field is the team-type DEFAULT a
new team inherits — it is not the records gate. The parity test locks
registry.competitive === isCompetitiveSlug(defaultTeamType) so the default and
the team-level truth can never silently disagree.
