P1 — REAL BUGS / EDGES (fix now)
------------------------------------------------------------------------------
  AUD-2  useRegisterIdentity AP#36 trio (#976). The guardians, player_guardians, and
    Promise.all(players, registrations) reads destructure data WITHOUT checking error.
    On ANY read error the child list silently empties -> the wizard falls back to
    manual entry -> the parent re-types -> the exact double-reg hole this feature
    closed. Also the open CI destructure-audit headroom. FIX: check error on all four.
  AUD-3  useRegisterIdentity guardian-missing-email edge (#976). If the authed
    parent's guardian row has no email (or no guardian row), AUTHED_INIT pre-fills an
    empty guardian, the guardian step is SKIPPED (authed entrySeq), and submit fails
    'guardian_email_required' with no way to correct. FIX: when guardian email is
    missing, force select:false (keep the full flow incl. the guardian step). Lanes G+H.
  AUD-4  Null-grade child wrongly shown "ineligible" (#976). SELECT_CHILDREN sets
    grade as `p.grade ?? ''`; an empty-string grade coerces to 0 in
    childIneligibleReason ('' < grade_min) -> a kid with no grade on file is disabled
    on the select screen even though the RPC would accept them. FIX: `?? null` in the
    reducer (childIneligibleReason already treats null as unknown). Lane G.
  AUD-5  AdminTeamsPage doesn't pass programType to TeamFormSheet (#977) -> the team
    3-state "Inherit" chip mislabels camp/tryout teams edited from that page as
    "Visible (season)". FIX: pass programType from the active season. Lane F.
  AUD-6  leagueAppsImport will zero-bill a future import (latent, from STEP-2 unify).
    The import writes financial_accounts.season_fee_cents but never posts a 'fee'
    transaction; under the unified model billed = SUM('fee'), so any future import
    batch bills $0. Harmless today (no import pending). FIX: post a 'fee' txn per
    imported account (as STEP-2 did for the 164). Lane D.
