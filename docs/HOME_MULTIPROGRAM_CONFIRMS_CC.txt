CONFLICT / FLAG (this is the one that shapes M-B)
------------------------------------------------------------------------------
events.team_id is NOT NULL. Every event must belong to a team. So M-B's
"reuse events for camp sessions" does NOT work as-is for a camp that has no
team — you can't insert a team-less event.

Two ways out, and one is clearly cleaner:
  (B-reuse, RECOMMENDED) Model a camp as a (program + season + TEAM) triple —
    the camp's "team" is its enrolled roster. Then camp SESSIONS are ordinary
    events under that team_id with event_type='camp'/'session', and EVERYTHING
    reuses with zero event-schema change: the team-scoped home hooks,
    check_ins (attendance), event_coach_assignments + coach_payouts (comp),
    even registrations.team_id already exists. The camp differs from an AAU
    team only in program_type + the enroll-once UX, not in storage.
  (B-relax) Add events.program_id + make team_id nullable for non-team
    offerings. Bigger blast radius — every team-scoped read (the whole home,
    schedule, records) would need a program-or-team branch. Not recommended.

So M-B's "reuse events first" is viable, but it implies camps get a team row.
Recommend the lane state that explicitly: camp = program(camp) + season +
team(roster) + events(sessions). If a camp must NOT have a team, then events
needs the program_id/nullable-team change and the cost jumps.
