DR-1 — THEME ACTUATION  (surfaced by S1 My Preferences, #915)
==============================================================================
REQUEST: theme is shipped as a control but is a DEAD COLUMN — decide whether to wire
  real dark mode or leave it stored-not-actuated.
EVIDENCE: verified — no dark stylesheet, no `@media (prefers-color-scheme)`, no
  `data-theme`/ThemeProvider, and user_preferences.theme is never READ anywhere in src/.
  Writing 'dark' has zero visual effect today. S1 persists the pref + says so in the
  Appearance help copy.
OPTIONS + REQUIREMENTS:
  (a) Leave stored-not-actuated. No further work; the pref is ready for a future track.
  (b) Wire dark mode. Requires: a full `--as-*` dark token set (a dark value for every
      §3 token), a theme actuator (apply data-theme/class on <html> from
      user_preferences.theme + a `system` media listener at app root), and a per-surface
      contrast/design pass. This is its OWN multi-PR arc + design lane — not part of the
      by-surface settings build.
CC LEAN: (a) for now. Dark mode is a deliberate design arc, not a settings byproduct;
  the control already persists intent for when you schedule (b).

DR-2 — DENSITY LEVELS (master-spec correction)  (surfaced by S1, #915)
==============================================================================
REQUEST: the render + master spec show 3-level density (Minimal/Medium/Maximum), but the
  shipped code is 2-state. Confirm 2-state (correct the spec) or restore 3-level.
EVIDENCE: useDensity.js: `VALID = ['minimal','maximum']`, per CLAUDE.md §16.2 (ratified
  2026-06-05, the 3→2 collapse). useDensity.setDensity THROWS on 'medium'. The DB default
  {"default":"medium"} is stale (useDensity ignores it → falls to 'minimal'). S1 built
  2-state and validates a stored 'medium' → 'minimal' on read.
OPTIONS + REQUIREMENTS:
  (a) Confirm 2-state; drop "Medium" from the master spec/render. No code change.
  (b) Restore 3-level. Requires reverting §16.2's collapse: re-add 'medium' to
      useDensity.VALID + NEXT_IN_CYCLE, re-audit every density consumer, and amend §16.2.
      Reopens a closed operator-ratified decision.
CC LEAN: (a) — matches shipped code + doctrine. Worth correcting the master spec so later
  density-touching surfaces don't re-assume 3-level.
