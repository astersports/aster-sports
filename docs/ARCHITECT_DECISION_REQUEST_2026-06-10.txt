--------------------------------------------------------------------------------
REQUEST: How does a parent register for a program type that has no divisions?
EVIDENCE (live + code):
  • The ONE live, published, registration-OPEN program is "Fall 2026 Tryouts"
    (slug fall-2026-tryouts, window 06-07 → 06-26, OPEN now), program_type=TRYOUT.
  • Divisions are SEASON-ONLY by design (programRegistry hasDivisions=false for
    tryout/camp/clinic) → this program has 0 divisions / 0 fees (verified live).
  • The funnel REQUIRES a division: RegisterFlowPage firstChild && !division →
    "This registration isn't available." NET: a parent who opens the live link
    today CANNOT register. The division-gated funnel can't serve non-season types.
  • The b3-funnel render already shows a tryout with a FLAT per-child fee
    ("Charlie · Fall 2026 Tryouts $45") and NO division picker — the intended UX
    already implies a flat-fee path.
OPTIONS:
  (A) Implicit flat-fee unit. Auto-create ONE "default division" per non-season
      program at create-time (gender=coed, no grade band, one base fee). Funnel
      works UNCHANGED; fees reuse division_fees. REQUIRES: a fee field on the
      non-season create path + funnel skips the visible division picker when there
      is exactly one implicit unit. Smallest delta; one funnel, one fee source.
  (B) Separate flat-fee path. Program-level fee + a register flow that skips the
      division step for non-season types. REQUIRES: new flow surface + a new
      program-level fee source + a second money path to reconcile against #63.
  (C) tryout_sessions/attendees. Parent picks a tryout session. REQUIRES: a
      different flow; tryout_sessions exist but are empty; does NOT match the
      b3-funnel render.
CC LEAN: (A). Makes the live program registrable with the least new surface,
  keeps ONE funnel + ONE fee source (preserves #63), matches the existing render.
RENDER NEEDED if A: non-season register flow (flat fee, no visible division pick)
  + the admin "tryout/camp fee" capture field. (= render G1.)
