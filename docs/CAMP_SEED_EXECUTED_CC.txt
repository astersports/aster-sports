3. ONE THING THE PLAN DID NOT ANTICIPATE (handled)
------------------------------------------------------------------------------
team_players carries an alignment-lock trigger (tg_team_players_alignment_check,
from migration 20260505201932_wave4_roster_alignment_lock): a status='active'
team_players row REQUIRES a matching roster_members (team_id, player_id) row
first. The first Lab attempt (team_players only) aborted on this — and rolled
back atomically (no orphan program/team). Resolution: the Lab seed inserts
roster_members (32) BEFORE team_players (32), in one atomic DO block with a
32/32 guard (RAISE EXCEPTION -> full rollback on any mismatch). roster_members
inserted with null sizes/jersey (the equipment-align trigger fires only on
size/jersey columns -> no-op; the roster_members alignment trigger fires only on
DELETE/UPDATE -> insert is clean). Note for the Fall Lab seed: same two-step
order (roster_members then team_players) is required.
