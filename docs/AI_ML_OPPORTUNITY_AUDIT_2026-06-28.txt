3. CROSS-CUTTING ARCHITECTURE NOTES
=====================================================================================
- ONE shared synthesis edge fn should back all "facts → org-voice prose" items (clone the
  proven suggest-briefing-closer scaffold: getAppSecret + callClaude + JWT/authz + AP#30
  mirror). Do not stand up four functions.
- ONE shared baseline snapshot table backs all trend/anomaly items — build once.
- ONE shared per-family risk-signal hook backs all financial-insight items (AP#42/#63 —
  one source, one scope, labeled).
- DETECTION ≠ NARRATION: keep detection deterministic/statistical (testable, free, no
  hallucination); reserve the LLM for the human-readable sentence.
- COST control: gate model calls on explicit action or threshold-cross + cache per
  signal-hash. NEVER call the model on the 60s alert poll.
- §16.12 enforcement point: a shared facts-allowlist that strips player_id-keyed game-stat
  fields before ANY AI call that touches game data.
- PRIVACY default: every synthesis call role-scoped server-side before facts leave the app
  (parent=own kids, coach=own team, admin=all).
