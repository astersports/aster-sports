=====================================================================
READ FIRST — TIERS vs RELEASES (rev 2 fixes the numbering collision)
=====================================================================
TIERS = what the CUSTOMER gets (access/price). RELEASES = the order WE build/ship.
The earlier "P0..P6" phase numbers collided with the tier numbers — renamed to Releases.

TIERS (the product):
  • TIER 0 — FREE HUB. Completely free. NO trial, NO card, no login required. Free forever.
    The first level every parent experiences (the acquisition gateway).
  • TIER 1 — HUB PLUS. The weekend-assistant layer. 7-DAY FREE TRIAL → $20/mo per account
    ($200/yr). A parent opts into this FROM the free Tier 0 Hub.
  • PROGRAM TIERS (B2B) — P0 claimed listing → P1 program hub → P2 registration (orgs).

RELEASE ORDER (free-first, matching the product story):
  R1 Free Hub (Tier 0)  →  R2 Hub Plus (Tier 1)  →  R3 Cutover/Host  →  R4 Program ladder
  →  R5 Premium/AI/Film.
Super-admin (Frank) operates from R1 onward (DR-R: super-admin first, restrict down).
Engineering note: the Tier-1 billing rails are independent and CAN be built/tested in
parallel during R1, but they do NOT go live to users until R2. So "free-first" is the
user/launch order; the rails can be proven early in test mode without affecting Tier 0.

=====================================================================
R1 — FREE HUB  (delivers TIER 0 — free, no trial, no card)
=====================================================================
GOAL: the consolidated FREE Hub live on astersports.app — any parent, no login. The wedge
and the FIRST level. Nothing here ever asks for a card.
FUNCTIONALITY (Tier 0, free forever):
  • Search any AAU team; tournament directory/browse; live scores; team schedule;
    standings/records/bracket; team detail; up-next countdown; constellation home; Ask-Scout.
  • Track teams: anonymous (localStorage); on optional sign-in, tracks SYNC to the account
    so a no-org parent persists across devices. (Sign-in is optional and free; not a trial.)
  • A signed-in parent with NO program lands on the family Hub (not a dead-end).
WHAT'S BUILT:
  • IN-APP (reuse): all `get_public_*` RPCs (shipped, anon-gated), `useWeather`,
    `getDirectionUrls`, the public-route pattern (`App.jsx:74-82`), `family_subscriptions`.
  • NEW: no-org family-Hub home route; no-org-tolerant `SeasonProvider`/`WeatherProvider`;
    DR-A role-type routing (anon→Hub, parent→family Hub, staff→role home); DR-P
    localStorage store + sync-on-login → `family_subscriptions` (public team refs only).
  • PORT (.tsx→.jsx) over existing RPCs: tracking, schedule, standings/bracket, find,
    constellation home, conflict radar (read), Scout (re-point SSE).
DEPENDS ON: nothing user-facing. (One tiny prep — add the `plan` column to
`family_entitlements` — can ride here or with R2; invisible.)
EXIT: an anonymous visitor uses the full free Hub on astersports.app; a no-org signed-in
parent persists tracked teams; zero org, zero card, ever, at this tier.
SCOPE: mostly routine UI port + one additive auth seam (architect-aware: the no-org route).

=====================================================================
R2 — HUB PLUS  (delivers TIER 1 — 7-day free trial → $20/mo)
=====================================================================
GOAL: the paid weekend-assistant a parent opts into FROM the free Hub. The money-path +
the assistant features ship together as one customer-visible upgrade.
FUNCTIONALITY (Tier 1, 7-day free trial → $20/mo per account):
  • Upgrade flow: "Start 7-day free trial" → assistant unlocks → $20/mo after 7 days,
    cancel anytime; explicit pre-charge consent; reminder BEFORE the first charge w/ one-tap
    cancel.
  • Assistant: leave-by + traffic-aware ETA; per-game weather across the list
    (+severe/feels-like); schedule-change + low-RSVP push alerts; conflict/"babysitter"
    radar incl. travel-time conflicts; whole-weekend multi-kid agenda; bracket scenarios;
    calendar sync (webcal); add-a-tournament (paste-to-track).
WHAT'S BUILT:
  • ENTITLEMENT BRIDGE: read `is_entitled` (exists; already counts `trialing`); `plan`
    column free/plus.
  • MONEY-PATH (NEW Supabase edge fns): `billing-checkout` (trial-mode-agnostic + `trial_end`;
    default card-up-front, DR-T) and `billing-webhook` (Stripe-signature-validated; SOLE
    writer of `family_entitlements`). Studio Stripe code as TEMPLATE. Trial-reminder cron;
    Stripe keys in `app_secrets`; customer portal.
  • ASSISTANT: wire `PlusGate`→`is_entitled`; NEW weekend agenda + travel-time conflicts +
    traffic ETA (DR-6/N1 vendor); push via the platform's `send-push`; ICS via team/family-feed.
DEPENDS ON: R1 (the Hub UI to gate). Rails can be pre-built/tested during R1.
EXIT: a parent starts the trial from the free Hub, uses the assistant, converts to $20/mo;
entitlement gates correctly; surprise-charge guards (consent + reminder) verified in test mode.
SCOPE: architect-scoped + onboarding-gate (money-path); owner-applied.

=====================================================================
R3 — CUTOVER & HOST
=====================================================================
GOAL: make astersports.app THE front door; retire the standalone; unify the host.
FUNCTIONALITY: no new user features — operational unification, no broken links.
WHAT'S BUILT: public Hub = the .app front door (DR-A); 301 `astersports.io/aau` + the
legacy-hoopers URL → the app; sitemap/canonicals; host swap Vercel→Railway (DR-H, atomic);
retire legacy-hoopers; slim astersports-web to Studio + landing (pointer PR #242).
DEPENDS ON: R1 (front door exists), ideally R2.
EXIT: one app, one front door, one host; old URLs redirect; traffic preserved.
SCOPE: architect-scoped (domain/deploy cutover).

=====================================================================
R4 — PROGRAM LADDER  (B2B — AAU programs sign on)
=====================================================================
GOAL: programs adopt the platform to maintain content + run registration.
FUNCTIONALITY:
  • Program-P0 claimed listing — a verified org claims its public Hub presence (likely free).
  • Program-P1 Program Hub — org maintains schedules/rosters/scores/announcements.
  • Program-P2 Registration + payments — registration/enrollment + fee collection.
WHAT'S BUILT: IN-APP — org `subscription_plan`, `RegisterFlowPage`, financials, comms engine,
coach/admin surfaces. NEW — program claim/verify flow; program subscription money-path
writer; wire registration charging; program onboarding (super-admin → org-admin → coach).
DEPENDS ON: R2 (entitlement pattern reused); parallelizable with R2-R3.
EXIT: a program self-onboards, maintains its schedule, runs a registration with payment.
SCOPE: architect-scoped (B2B money-path + tenancy). Charging defined as we progress.

=====================================================================
R5 — PREMIUM · AI · FILM  (last)
=====================================================================
GOAL: the evergreen/premium layer; biggest net-new build, gated.
FUNCTIONALITY: family film room (clips / AI auto-highlights), COPPA-gated (`can_access_child`,
guardian-verified — locked for everyone incl. super-admin until verified); advanced AI
recaps; Tier-1+ add-ons (bundle vs add-on TBD).
WHAT'S BUILT: NEW media pipeline (Supabase Storage), capture/clip, AI auto-highlight; COPPA
consent flow; premium gating. Latest AI models throughout (DR-AI), no-fabrication guardrail.
DEPENDS ON: R2 (entitlement) + COPPA verification flow.
EXIT: guardian-verified parents access film; COPPA boundary holds at the DB.
SCOPE: architect-scoped (COPPA + storage + money).
