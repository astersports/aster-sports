PILLAR HEADER

REPO / BRANCH: astersports/aster-sports @ main
WHAT THIS PILLAR IS FOR: the multi-tenant youth-sports management SaaS —
  schedules, rosters, RSVPs, team messaging/briefings, financials, and AAU
  tournament ingest. Pilot tenant: Legacy Hoopers.
HARD INVARIANTS THAT APPLY (all of them apply on this pillar):
  · MONEY PATH — the ledger is the sole source of balance; no bill-for-a-no-op;
    writes must be idempotent. (Today: NO Stripe; admin-hand-entered.)
  · CHILD DATA / COPPA — guardian-verified gate; a child's row stays false-by-
    default to anyone but a verified guardian + org staff.
  · TENANT ISOLATION — one org can never read/spend another's data.
  · HONEST CLAIMS — UI/labels must never contradict the data.
  · SCHEMA MIGRATIONS — owner-applied only; CC mirrors, never applies.
  · AI DETERMINISM QUARANTINE — generative output stays out of the money/
    schema/child paths; facts in, prose out, admin-gated.

— MONEY PATH —
M2. FIX THE OVERDUE PREDICATE (never-paid debtors) + one shared overdue source.
  Where: src/lib/alerts/queries.js:117. Win: the worst debtor class stops being
  invisible; the admin lane + the alert stop contradicting on one screen (honest-
  claims). Cost: very low (read/alert logic only; no ledger write). Privacy: none.
  GATE 🟢 ROUTINE. Collision: none. LEAN: SHIP FIRST (live bug).

M1. AUTOMATED LEDGER-RECONCILIATION CHECK (SUM(transactions)==balance + void-pair
  integrity). Where: behind useSeasonFinancials.js / family_balances; a vitest
  fixture + periodic SQL check. Win: catches a corrupted void-pair or a BYPASSRLS
  mutation before a parent sees it; fills the §6 reconciliation invariant that has
  no automated guard. Cost: low (read-only). Privacy: none. GATE 🟢 ROUTINE (read-
  only verification). Collision: none — it ENFORCES the money invariant. LEAN: PILOT→ship.

M3. STRIPE (or Connect) + webhook-sole-writer ingestion. Where: replaces the
  admin-hand-entry model (RecordPaymentForm, submit_registration). A stripe-
  webhook-receiver edge fn becomes the SOLE automated writer of `payment` rows,
  idempotent on event.id/payment_intent.id (model it on the existing app_secrets +
  verify_jwt=false pattern, AP#31/#33). Win: online registration checkout;
  eliminates manual double-submit risk. Cost: HIGH — PCI scope, refund/dispute
  reconciliation, signing-secret handling. Privacy: new sub-processor (Stripe);
  card data tokenized client-side (never touches our DB). GATE 🔴 HUMAN (money
  path; Architect-scoped + Frank confirm). Collision: redefines "who writes the
  ledger" — hand-entry must reconcile against the webhook. LEAN: PROTOTYPE in a
  test-mode harness; ratify before any prod key.

M4. SERVER-SIDE void/charge via SECDEF RPC + idempotency key. Where: the 3 raw
  client inserts (RecordPaymentForm.jsx:44, AddChargeSheet.jsx:23, void :39). Win:
  enforces void-pair field integrity at the DB; true idempotency; removes the last
  raw client writes to the ledger. Cost: medium (3 RPCs + caller migration). GATE
  🔴 HUMAN (changes who writes the ledger). LEAN: PROTOTYPE; pairs with M3.

M5. INVOICE PDF + dunning (the invoices table + ar_aging view ALREADY exist,
  20260613203959 / 20260613210010). Where: dormant invoices substrate + parent
  receipt gap. Win: parent-facing receipts; automated overdue follow-up off
  ar_aging buckets. Cost: medium (server-side PDF render — keep off the 350KB
  bundle; reminder cron on the existing briefing-auto-draft-tick scheduler). GATE
  🟡 — PDF/read is routine, but AUTO-SENDING dunning emails collides with the
  Stream-B "auto-DRAFT only, never auto-fire" doctrine (§16.5): keep draft-only
  (🟢) or gate any auto-send (🔴). LEAN: PROTOTYPE draft-only.

— PRIVACY / ISOLATION —
P1. AUTOMATED CROSS-TENANT + CROSS-CHILD ISOLATION TEST HARNESS. Where: the whole
  RLS surface (current_user_player_ids, _teammate_player_ids, registrations PII

  NEGATIVE (tenant A reads 0 of B; parent A reads 0 of child B), runs in CI. Win:

P2. RLS ADVISOR / LINTER IN CI. Where: supabase/migrations/*. Scheduled get_advisors

  flagged at author time. Cost: low. GATE 🟢 ROUTINE. Collision: none. LEAN: ship.

P4. COPPA / PHOTO-VIDEO-RELEASE CONSENT FLOW + enforcement. Where: consentVersions

  VERSIONS, wire the signup-accept UI + a re-affirm gate on version bump. Win:
  legal/COPPA defensibility for youth PII; the natural gate any future media/film

P3. IMPERSONATION / VIEW-AS AUDIT LOG. Where: useHomeRole.js / RoleSwitcherSheet.
