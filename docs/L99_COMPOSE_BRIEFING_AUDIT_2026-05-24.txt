Yesterday's wizard send error `TypeError: Cannot read properties of
undefined (reading 'from')` was root-caused via this audit's Agent 2

    const { supabase } = await import('../supabase');

  Under PWA stale-cache conditions (per STATE_OF_AFFAIRS_L99_v6 §3.3
  + yesterday's adjacent SW "Failed to convert value to Response"
  error), the Vite chunk for '../supabase' resolved malformed.
  Destructure made supabase undefined; line 78
  `supabase.from('comms_message_recipients').insert(...)` threw
  TypeError "Cannot read properties of undefined (reading 'from')"
  — exact match.

Fix shipped via PR #493 (static imports at top of file). After
Vercel deploy, Frank's coach_roundup wizard send should succeed
(symmetric across all 4 composerSubmit-path kinds — family_guide,
game_recap, tournament_prelim, tournament_recap, schedule_change
all flow through queueComposedMessages).

Investigation discipline note: PR #491 (yesterday's earlier fix
addressing slice.kind contract gap) was a real fix for a real bug,
but NOT the user-visible error. The premise that slice.kind throw
was the visible error was inferred from section_count=0
correlation, not verified against literal error text. Memory
`distinguish-inferred-vs-confirmed-error-source` saved at session
close yesterday captures this discipline gap.
