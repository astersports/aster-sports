2. R-1 , POPOVER SURFACE PRIMITIVE
------------------------------------------------------------------------------
   RULING: (a) lightweight inline popover. Approve CC lean.
   - BottomSheet / FullScreenForm are the FORM vocabulary (AP#15). This payload is
     a read-only disclosure with zero controls. Routing it through a form primitive
     overloads the vocabulary and drops a heavy surface on a two-line payload.
   - Do NOT create a new shared <Disclosure> primitive. One consumer does not earn
     an abstraction. Promote at the third instance, not the first. Keep the popover
     local to WeatherPopover.jsx.
   - Anchored to the trigger, opens below, flips above if it would clip the
     viewport bottom on mobile. No backdrop dimming (it is a glance, not a modal).

------------------------------------------------------------------------------
3. R-2 , POPOVER CONTENT
------------------------------------------------------------------------------
   RULING: one glanceable card, three lines max.
     Line 1: [Lucide condition icon] + condition label (e.g. "Partly cloudy")
     Line 2: matched-hour temp, the same value already on the rail, large
     Line 3: day rain chance, "{n}% chance of rain"
   - Reject the surrounding-3-hours option. That is a forecast strip, too much for a
     delight glance and it pushes toward edge-of-coverage fabrication.
   - If day rain% is unavailable (hourly-only path), OMIT line 3. Never render "0%".
   - No location string in this surface. A null location renders nothing here.

------------------------------------------------------------------------------
4. R-3 , POPOVER TRIGGER + A11Y
------------------------------------------------------------------------------
   RULING: the existing rail indicator (EventCard {icon}{temp} on the facts line)
   becomes the trigger. Keep it where it is. Parents' eyes already land there.
   - Hit area >=44px via invisible padding around the small visible glyph.
   - It is a <button> (focusable, Enter/Space activates), not a div with onClick.
   - aria-label: "Weather for {event title}: {label}, {temp} degrees". Spell out
     "degrees", screen readers mangle the ring glyph.
   - Dismiss on outside-tap and Esc. Return focus to the trigger on close.
   - When there is no forecast for the row, render nothing (no trigger, no empty
     ring). The trigger only exists when there is something to disclose.

------------------------------------------------------------------------------
5. R-4 , RAIN BANNER TONE + TOKEN (re-grounded)
------------------------------------------------------------------------------
   RULING: INFO tier. Frank's "friendly heads-up, not alarm" = info tier.
   - Accent (icon, and an optional 3px left border): the info color.
   - Soft background: the info soft background.
     [CC VF-1 concrete tokens: background var(--as-info-soft); accent var(--as-info);
     reuse SEVERITY_TOKENS.info from src/lib/alerts/severityTokens.js. Do NOT use
     --as-cobalt / --as-card / color-mix (they do not ship).]
   - Icon: Lucide cloud-rain (or umbrella), stroke-width 1.75.
   - Reject amber (the WARN tier) and red (critical) — they read as alarms.
   - Copy tone: friendly, plain, action-light, no alarm words. No em dashes.
     Approved templates (CC picks one, keeps the day named):
       "Looks like rain Saturday. Might want to leave a few minutes early."
       "Rain likely Saturday. Pack a jacket for the trip."
     Banned: "PRECIPITATION ALERT", "WARNING", anything all-caps or urgent.

6. R-5 , RAIN BANNER PLACEMENT + THRESHOLD + DISMISSAL
------------------------------------------------------------------------------
   RULING:
   - Surface: HOME only. Schedule is a scanning surface where a banner competes with
     the list. Home is the "what's coming up" surface where a single heads-up belongs.
     (Frank may override to "both"; if so, schedule copy attaches to the next-event
     header only, never repeated per row.)
   - Scope: the NEXT upcoming event only. One banner, one event, one heads-up.
   - Threshold: precipitation_probability_max >= 50 for that event's date.
     Named constant RAIN_BANNER_THRESHOLD_PCT = 50 in lib/constants.js.
   - Dismissal: session-dismissible via sessionStorage, keyed by event id.
     Key: rainbanner:dismissed:{eventId}.
   - Never fabricate: no daily coverage, no coords, or fetch fail => no banner.
