§1 WHAT WAS BROKEN (recap, one paragraph)
----------------------------------------------------------------
Since migration 20260528140000, the four *_select_public RLS policies
gated anon reads on `org_id IN (SELECT id FROM organizations WHERE
public_listing_enabled = true)`. Policy USING expressions evaluate as
the CALLER; organizations has no anon-readable policy; the subquery
returned empty for anon, permanently. Verified pre-fix as anon: 0 teams,
0 events. Stacked on that (AP #26 two-cause pattern): PublicSchedulePage
and the team-feed ICS edge function both selected a nonexistent
`events.location_name` column (42703) — the page swallowed the error
(AP #36) and rendered a false "No upcoming events"; the feed checked it
and 500'd. Net: QR links, shared schedules, and calendar subscriptions
all dead for ~2 weeks. Inverse leak: game_results_select_public had no
org gate (anon read all 91 published results); anon also held
write-class grants on all 12 schedule tables, fail-closed only by RLS.

§9 DEVIATIONS / EXTENSIONS vs THE RULED SHAPE — FOR RATIFICATION
----------------------------------------------------------------
 D1 SELECT revoked ENTIRELY on the 8 no-public-need tables. The ruling's
    STEP 1 text named only teams/locations column-narrowing; the "et
    al." principle (anon sees only what public render needs) extended
    naturally to tables with ZERO public-render need and ZERO anon
    policies (verified before revoking). Strictly tightening; nothing
    anon-side reads them; authed grants untouched.
    RISK: none identified. ASK: ratify.
 D2 get_public_subscribe_info() is NEW (not in the ruling). Forced move:
    the ruled token-column revoke would have re-broken the public page's
    Subscribe (it bulk-selected teams.team_feed_token). The RPC serves
    the token one-team-at-a-time, only for public-listed orgs. CAVEAT
    stated plainly: for a PUBLIC-LISTED org, anyone holding a team id
    can fetch that team's feed token — which is exactly the page's
    intended function (the feed is the thing being publicly offered);
    what's closed is bulk enumeration and any access to non-listed
    orgs' tokens. Token rotation (20260602225959 annual) unaffected.
    ASK: ratify, or rule a stricter shape (e.g., short-lived signed
    subscribe URLs) for the SD-16 family-feed arc to adopt.
 D3 game_results gate composes through PARENT-EVENT VISIBILITY
    (EXISTS events as the caller) rather than the literal
    events->teams->org_is_public_listed traversal in the ruling text.
    For anon the outcomes are identical (anon's event visibility IS the
    public-listing gate). For authenticated it is STRICTER than ruled:
    cross-org authed users lose the read (they can't see the parent
    event) — closing a leak the review hadn't even graded. Org members
    and parents unaffected (their event visibility covers them; plus
    game_results_select_staff). Chosen for single-source-of-truth: the
    results gate can never drift from the events gate.
    ASK: ratify the composition (it's the AP #63-resistant shape).
 D4 events_select_public composes through teams-as-caller (EXISTS teams
    t WHERE org_is_public_listed) — the teams row must be anon-visible
    via teams_select_public, which holds by construction. Equivalent to
    the literal form under current policies; noted for completeness.
 D5 Write-class revoke included DELETE, TRUNCATE, TRIGGER beyond the
    ruled INSERT/UPDATE/REFERENCES — anon held all six (discovered at
    execution); revoking the worse three is unambiguously in-intent.
    ASK: ratify (formality).
 D6 eventTitleAudit allowlist relocation (mechanical consequence of the
    ruled _helpers.ts/icsCore split; the audit's intent preserved —
    icalHelpers.js itself is no longer allowlisted).
 D7 PROCESS: the MCP deploy default flipped verify_jwt=true on v9
