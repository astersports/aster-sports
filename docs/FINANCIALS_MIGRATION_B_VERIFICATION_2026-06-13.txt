B-2 [P1 — applies but WRONG, live-proven] ar_aging mis-buckets never-paid debt
──────────────────────────────────────────────────────────────────────
Spec §4B.5:
    WHEN now() - COALESCE(fb.last_payment_at, now()) < interval '31 days' THEN 'd0_30'
For a family that was billed and has made ZERO payments, last_payment_at IS NULL →
COALESCE substitutes now() → now()-now()=0 → buckets as 'd0_30' (freshly current). The
worst debtor class (never paid a cent) renders as the NEWEST bucket.
LIVE PROOF (simulated the exact CASE against family_balances today):
    Jon Zinman   $1,275  last_payment 2025-09-15  → d60_plus     (correct; he paid once)
    Frank S.     $70     last_payment NULL          → d0_30  ← WRONG, never paid
This is the SAME blind-spot class as the just-merged audit's P1-4 (the overdue ALERT's
.lt('last_payment_at') silently drops NULLs). The elite layer re-introduces it in the AR view.
PATCH: age never-paid debt from when it was billed, not from now(). Once invoices exist
(Migration A), use the invoice issued_at; until then, financial_accounts.created_at:
    -- replace COALESCE(fb.last_payment_at, now()) with COALESCE(fb.last_payment_at, fa.created_at)
    -- joining financial_accounts fa ON fa.id = fb.account_id
  so a never-paid balance ages from the billing date and lands in the correct bucket.
  (Equivalently: NULL last_payment_at → oldest bucket.) Lock with a test row.
