2. COACH COMP CARD (D-A)  —  [DATA-GATED] + one card to build
------------------------------------------------------------------------------
Admin-side payout infra ALREADY EXISTS: coach_payouts table, CoachPayoutsSection.jsx
(inside the financial dashboard), useCoachPayoutsPending.js. The 11
coaching_assignments rate rows exist (coach × team × per-session rate). What's
missing is the data that feeds payouts AND the coach-facing card.

  Missing attribute 1 — event_coach_assignments (0 rows):
    WHO coached WHICH event. The keystone — without it there is nothing to
    accrue a payout against.
    Required: a capture path. Either (a) an admin "assign coach to event"
    action on the event, or (b) derive the assignment from who runs check-in
    at the event. Until rows exist, no accrual is possible.

  Missing attribute 2 — coach_payouts (0 rows):
    the accrued amounts. Required: an accrual step that, when an assigned
    session completes, writes one payout row = coaching_assignments rate ×
    session, status 'pending'. Depends on attribute 1. Can be a DB trigger
    (auto-accrue on session completion) or an admin action (review-then-accrue).

  Missing build — the COACH'S OWN comp card:
    CoachPayoutsSection is the ADMIN view. The coach-home comp card + a
    useCoachComp hook (this coach's pending/paid totals) is NOT built
    (deferred in CoachTail).

  Migration: probably none (tables exist); ONE trigger/function only if you
  want auto-accrual rather than a manual admin step.
  DECISIONS:
    D-A1: How are coaches assigned to events — explicit admin action, or
          derived from check-in? (Sets where event_coach_assignments comes from.)
    D-A2: Accrual = automatic (trigger on session-complete) or admin
          review-then-mark? (Auto = a migration; manual = no migration.)
    D-A3: Coach-home comp card scope — just "you're owed $X (N sessions
          pending)" tapping to detail, or a fuller breakdown?
