[1] pay_coach TOCTOU + blind p_amount_cents (HIGH, latent — needs concurrent
    admins; LH has 1-2). You own pay_coach and just verified it in PR-2 review,
    so this rewrite is yours to apply + verify. The bug: pay_coach INSERTs the
    payout (with the caller's p_amount_cents) BEFORE the UPDATE, and the UPDATE
    is `WHERE id = ANY(p_session_ids) AND pay_status='owed'`. A race where the
    sessions were settled between validate and update flips 0 rows but still
    commits a full-amount paid payout → phantom double-pay. Also the amount is
    trusted blindly (never recomputed from the flipped sessions).
    FIX (UPDATE→RETURNING first, abort if short, INSERT amount from flipped rows):

    CREATE OR REPLACE FUNCTION public.pay_coach(
      p_org uuid, p_coach uuid, p_season uuid, p_session_ids uuid[],
      p_amount_cents int, p_status text, p_method text, p_paid_at timestamptz, p_notes text
    ) RETURNS uuid LANGUAGE plpgsql SECURITY DEFINER SET search_path = public, auth AS $fn$
    DECLARE v_payout uuid; v_flipped uuid[]; v_sum int;
    BEGIN
      IF NOT user_has_role_in_org(p_org, ARRAY['admin']) THEN RAISE EXCEPTION 'not authorized'; END IF;
      IF p_status <> 'paid' THEN RAISE EXCEPTION 'pay_coach settles paid sessions only'; END IF;
      -- Flip first; only the coach's owed, in-org sessions flip. RETURNING gives
      -- us exactly what settled.
      WITH flipped AS (
        UPDATE event_coach_assignments ea SET pay_status='paid'
        FROM events e JOIN teams t ON t.id=e.team_id
        WHERE ea.id = ANY(p_session_ids) AND ea.event_id=e.id
          AND ea.coach_user_id=p_coach AND ea.pay_status='owed' AND t.org_id=p_org
        RETURNING ea.id, ea.pay_cents
      )
      SELECT array_agg(id), COALESCE(sum(pay_cents),0) INTO v_flipped, v_sum FROM flipped;
      IF v_flipped IS NULL OR array_length(v_flipped,1) <> array_length(p_session_ids,1) THEN
        RAISE EXCEPTION 'settlement aborted: % of % sessions were not settleable (already settled, not owed, or not yours)',
          array_length(p_session_ids,1) - COALESCE(array_length(v_flipped,1),0), array_length(p_session_ids,1);
      END IF;
      INSERT INTO coach_payouts (org_id, coach_user_id, season_id, amount_cents, status,
        payment_method, paid_at, source_assignments, notes)
      VALUES (p_org, p_coach, p_season, v_sum, 'paid', p_method, p_paid_at, v_flipped, p_notes)
      RETURNING id INTO v_payout;
      UPDATE event_coach_assignments SET settled_by_payout_id=v_payout WHERE id = ANY(v_flipped);
      RETURN v_payout;
    END $fn$;

    Note: amount is now v_sum (server-recomputed), not p_amount_cents — the param
    becomes advisory. The whole function is one tx, so the abort rolls back the
    flip. Client (PayCoachSheet) needs no change; it already sends the matching set.
