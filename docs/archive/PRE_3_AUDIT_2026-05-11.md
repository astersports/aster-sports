### Finding PRE3-2 — `useTournamentBriefing.js` would throw if it were reachable

Independent of the orphan finding: `useTournamentBriefing.js:86` calls `compose({ kind: 'tournament_prelim', data })`. The `KIND_COMPOSERS` registry in `engine/composer.js:76–81` deliberately removed `tournament_prelim` in Wave 4.2-A-8a:

```javascript
// Wave 4.2-A-8a: legacy compose() entries removed for the 4 kinds
// that now dispatch via RESOLVER_REGISTRY through composerSubmit
// (game_recap, tournament_prelim, tournament_recap, schedule_change).
const KIND_COMPOSERS = {
  academy_callup_notice: composeAcademyCallupNotice,
  weekly_digest: composeWeeklyDigest,
  announcement: composeAnnouncement,
  custom_message: composeCustomMessage,
};
```

If anything called `compose({ kind: 'tournament_prelim' })` today it would throw:
> `No engine composer for kind "tournament_prelim". Supported kinds: academy_callup_notice, weekly_digest, announcement, custom_message.`

Nothing does, because the orphan is unreachable. But this is a latent footgun if anyone reaches for `useTournamentBriefing` for "rendering tournament prelim" they would discover only after deployment.

   FROM comms_message_recipients
