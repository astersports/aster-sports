ITEM 8: RSVP coverage threshold gate
---------------------------------------------------------------------------
File: supabase/functions/briefing-auto-draft-tick/_handlers.ts
       (handleRsvpLow24h rewrite, +18 LOC)

Handler entry now reads org's nudge_rules.rsvp_coverage_threshold
(default 0.7). Per-event:
  1. Fetch total = team_players WHERE team_id=event.team_id AND status='active'
  2. If total === 0: skip with "no_active_roster"
  3. Fetch responded = DISTINCT player_id FROM event_rsvps WHERE event_id=e.id
  4. coverage = responded / total
  5. If responded > 0 AND coverage >= threshold: skip with "coverage_met"
  6. Else: nudge (covers cold-start: responded=0 and gap case)

DEFERRED (Wave 4.4): academy_callup_player_ids[] inflates effective
roster. v1 ignores callups in denominator (slightly underestimates;
threshold gate still functions). Document as future refinement.

MIGRATION REQUIRED (drafted in PR description for chat-side apply):
  UPDATE organization_settings
  SET nudge_rules = nudge_rules || jsonb_build_object(
    'rsvp_coverage_threshold', 0.7
  )
  WHERE NOT (nudge_rules ? 'rsvp_coverage_threshold');
