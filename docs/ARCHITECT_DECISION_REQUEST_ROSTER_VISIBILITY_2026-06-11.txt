RV-1 — MECHANISM: RLS-enforced or app-layer?
------------------------------------------------------------------------------
  REQUEST: is roster masking enforced in RLS or only hidden in the UI?
  OPTIONS:
    (A) RLS-enforced: exclude hidden teams' rosters inside
        current_user_teammate_player_ids() (and thus from the players RLS the parent
        sees). REQUIRES: the fn joins team -> program, drops teams whose effective
        roster_visibility is false. The data never reaches the client.
    (B) App-layer only: fetch all teammates, hide in the component. REQUIRES: nothing
        in RLS; trivially bypassable via the API. Leaks the names to anyone reading
        the network response.
  CC LEAN: (A) RLS-enforced. This is privacy; app-layer hiding ships a leak as a
    fix (the §16.7.1 two-surface lesson). The own kid stays visible via
    players_select_parent regardless, so a parent on a hidden camp still sees their
    own child, just not the other 31.

RV-2 — PLACEMENT / GRANULARITY: program, team, or both?
------------------------------------------------------------------------------
  REQUEST: where does the setting live? (Frank said "program AND team setup.")
  OPTIONS:
    (A) Program-level only: programs.roster_visibility boolean; all teams inherit.
        Simplest; no per-team control.
    (B) Program default + team override: programs.roster_visibility (default by type)
        + teams.roster_visibility_override (nullable; NULL = inherit program).
        Effective = COALESCE(team.override, program.roster_visibility). REQUIRES: two
        columns + the COALESCE in the fn + two UI toggles.
    (C) Team-level only: teams.roster_visibility boolean. No program default; admin
        sets each team. Misses the "set it once for the whole tryout" ergonomic.
  CC LEAN: (B). Matches Frank's "program and team" ask: the program sets the sensible
    default for all its teams (one switch for a whole tryout), the team override
    handles the exception (e.g. one camp team you DO want visible). NULL-inherit keeps
    the common case zero-touch.
