================================================================================
2. ROOT CAUSE — COURT NULL (state discovered by triage; not in the original 3-state model)
================================================================================
- division_games.source_facility_ref is an OPAQUE TM facility id (e.g.
  h202606041542226344c6972f899ba43), NOT human text. Westchester Summer League has TWO
  such ids (96 + 73 games) both resolving to the single "House of Sports" building venue.
  Those two ids ARE Court 1 and Court 2 — the venue-dedup ("dedupe_venues_to_buildings")
  correctly collapsed them to the building for directions, but that dropped the court label.
- The facility-id -> "Court N" map is NOT recoverable in-DB (the venue's tm_venue_key does
  not equal either facility id; the court names lived only in the TM Complexes panel).
- BUT the human location cell (g.location = "4 - House of Sports - Court 1") IS available at
  ingest time: it's held transiently in __location, used to resolve the building venue via
  cleanPlaceName (which strips the ordinal + "- Court X"), then discarded. The COURT is
  exactly the "- Court X" suffix cleanPlaceName strips. So court parses from g.location with
  NO facility panel and NO re-scrape beyond the ingest that already runs.

FIX A — GEOCODE BACKFILL (DONE, 2026-07-01) — SAFE, venues-only, no game touch
- Invoked geocode-venues edge function (idempotent, atomic SKIP-LOCKED claim, confidence
  gate; writes ONLY venues.lat/lng, never games/scores). 3 batches of 25 via pg_net
  (ingest_secret read server-side, never echoed).
- Result: venues 43 ok / 61 pending / 1 failed  ->  101 ok / 0 pending / 4 failed.
  58 venues newly geocoded; Apple Maps now resolves for those games.
- The 4 'failed' are low-confidence rejects the gate refused (never a wrong pin): they have
  spelled-out street numbers ("ONE Cross Road", "One Mount Hope Boulevard") Google returns
  as APPROXIMATE. They fall back to address/name links. Separately fixable (normalize
  "One"->"1"); not blocking. Venues: Marian Athletic Center, OLPH-Ardsley,
  St Matthew-Hastings HS2, St Matthews-Hasting HS1.

FIX B — COURT SPLIT (PREPARED as a reviewable PR; NOT run against live data yet)
- parseCourt(location) added to BOTH mirror files in one commit (AP #30):
  src/lib/aau/parseTournament.js (vitest source of truth) + the Deno mirror _parse.ts.
  Returns "Court 1"/"Court 2A" from the "- Court X" suffix, null for single-court gyms.
  Symmetric with cleanPlaceName: it keeps exactly what cleanPlaceName strips.
- index.ts sets court: parseCourt(g.location) on the game row. ADDITIVE — venue resolution
  is unchanged (still the deduped building), so the "fragment one gym into many venues"
  risk the original TODO worried about does NOT apply.
- Idempotent + scoped by external_game_id: the games upsert is onConflict
  (tournament_division_id, external_game_id), so re-ingest UPDATES court in place. A live
  tournament backfills court automatically on the next 5-min poll — no separate re-scrape.
  Past (non-polled) tournaments would need a one-time scoped re-ingest per tm_id_tournament
  if we want their court filled; that's an operator/architect call, kept OUT of this PR.
- CI-gating test: 10 parseCourt cases added to parseTournament.test.js (21 pass locally).
